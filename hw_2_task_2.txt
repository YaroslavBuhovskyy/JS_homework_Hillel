   'number' + 3 + 3
+ є ліво-асоціативним оператором, намбери перетворяться на стрінги з подальшою їх конкатенацією 
результатом буде стрінга 'number33'

    null + 3
тут null перетвориться на 0, а оскільки далі слідує Number, 
то відбудеться звичайне додавання, 
результатом буде намбер 0+3=3

    5 && "qwerty"
AND при порівнянні спочатку пройде по намберу і отримає true
але оскілька стрінга не є пустою, то і вона буде true
резульатом має бути останній true операнд 'qwerty' (правий, якщо обидва true)    

    +'40' + +'2' + "hillel";
+ перед стрінгами перетворює останні на тип Намбер, тому 40+2=42,
але оскільки останній операнд є стрінгою, то 42 перетвориться на стрінгу, далі відбудеться конкатенація
результатом буде стрінга '42hillel'

    '10' - 5 === 6;
тут буде false, 
оскільки потрійне = порівнює не тільки значення, а й типи
зліва рядок має числове значення, тому перетвориться на Намбер. 
10-5=5, але значення 5 != 6

    true + false
відбудеться приведення булевих типів до числових
1 + 0 = 1

    '4px' - 3
оскільки стрінга не може перетворитися на число, 'px' не зможе бути Number
відповідно на першому ж операнді відбудеться перетворення на NaN

    '4' - 3
а тут стрінга '4' зможе перетворитися на намбер 4
4-3=1, результатом буде 1

    '6' + 3 ** 0;
** є право-асоціативним оператором, тому спершу відбудеться піднесення 3 до 0 степеня, 
з математики це буде дорівнювати 1, а далі знову зліва стрінга перетворить намбер 3 на стрінгу
'6'+'1'='61'

    12 / '6'
буде 2, оскільки стрінгу 6 можна перетворити на число, зліва на право 12/6=2

    '10' + (5 === 6);
тут цікаво, оскільки прирівняння типів і значень 5 === 6 дасть false, але стрінга стоїть попереду, то
зліва на право стрінга перетворить false на стрінгу
'10' + 'false' = '10false'

    null == ''
буде true, так як з теорії обидва значення є фолсі, а оператор == порівнює значення

    3 ** (9 / 3);
27, тут все з матемитики, спочатку 9/3=3, а потім 3 підносимо до отриманого степеня

    !!'false' == !!'true'
1) !!'false' - це стрінга, яка не є пуста, тобто true, a оператор ! це заперечення, тобто НЕ true = false, 
  але далі йде знову !, тобто !false стає знову true, отже зліва буде true
2) !!'true' - НЕ ЗВЕРТАТИ УВАГУ:) на слово, тут все те ж саме, справа буде true
  true == true, значення однакові, тому результатом буде true

    0 || '0' && 1
спершу відповів true
спочатку відбудеться порівняння &&, далі з теорії повернеться праве, якщо обидва true
0 є фолсі, тому при операторі OR повернеться перше true 
але після перевірки в консолі зрозумів чому 1 (порівнюються ж значення, а не типи)

    (+null == false) < 1;
1) +null перетвориться у Намбер, тобто 0
2) 0 є фолсі, тому оператор ==, порівнявши обидва значення false, поверне true
3) а true в числовому значенні = 1, але 1 < 1 - це false

    false && true || true
1) AND, порівнюючи false та true, поверне false
2) OR, порівнюючи отримане в 1) false та true, пвоерне true 

    false && (false || true);
1) як і в математиці, спочатку відбудеться дія в дужках; OR поверне true (якщо воно є хоч одне)
2) AND поверне false, тобто результатом буде false

    (+null == false) < 1 ** 5;
1) 0 == false, це фолсі значення, тому буде true
2) 1 у будь-якій степені буде 1, тобто true
3) але 1 !< 1, тому результатом буде false
